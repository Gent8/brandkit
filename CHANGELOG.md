# Changelog

All notable changes to brandkit are recorded here. The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) and the project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.3.0] ‚Äî 2026-05-04

### Added

- **`brandkit trim`** ‚Äî new CLI subcommand that rewrites the root `viewBox` of an SVG to the rendered geometry bbox so the mark fills its canvas at every output size. Vectorizers (Recraft, vectorizer.ai) emit marks padded inside a much larger square; trim closes that gap so favicons, og-images, and print all render tight instead of inheriting 40‚Äì60% empty padding. Pure SVG-in / SVG-out, idempotent. `--strip-bg` also drops vectorizer-added canvas-fill rects. Wired into `brandkit gen` automatically; standalone for SVGs that came from elsewhere.
- **`brandkit_trim` MCP tool** ‚Äî same surface over MCP. Inputs: `svg`, `padPct?`, `stripBackground?`, `keepDimensions?`. Returns trimmed SVG as text.
- **`computeBbox(svg)`** export from `src/trim.js` ‚Äî reusable geometry-bbox helper that handles `<path>`, `<rect>`, `<circle>`, `<ellipse>`, `<line>`, `<polygon>`, `<polyline>`, per-element transforms (translate, scale, matrix), stroke widening, and skips invisible elements + `<defs>` / `<clipPath>` / `<mask>` / `<symbol>` blocks.
- 8 new smoke tests covering trim's bbox computation, idempotency, viewBox rewriting, `keepDimensions`, and composition with `verify`.

### Changed

- **License: AGPL-3.0-or-later ‚Üí Apache-2.0.** Permissive, commercial-friendly, no copyleft. If you previously avoided brandkit because of AGPL, this is for you.
- **`brandkit gen` survivors are now trimmed** by default (with `stripBackground: true`). Existing pipelines see no API change; outputs just stop carrying vectorizer padding into downstream rasters.
- **`brandkit_recolor` MCP tool description** clarifies it's pure text-in / text-out and that hsl()/named colors are rejected upstream.
- README repositioned around the CI-gate framing (*"Brand drift as a build failure"*) and updated to reference top-level `media/` paths instead of `website/`.

### Fixed

- Favicons and og-image rasters generated by `brandkit export` now fill their canvas ‚Äî previously the mark inherited the input SVG's padding, producing visibly small/offset 16√ó16 / 32√ó32 icons. `brandkit trim` closes the gap; `brandkit gen` does it automatically.

## [0.2.0] ‚Äî 2026-05-03

### Added

- Initial public release.
- `brandkit recolor` ‚Äî snap every `#hex` and `rgb()` in an SVG to the nearest brand-palette color.
- `brandkit verify` ‚Äî CI gate: exit 0 if every color is in the palette, exit 1 with a list of offenders otherwise.
- `brandkit export` ‚Äî render the full asset bundle (16/32/48/128/180/192/512 PNGs, multi-res favicon.ico, apple-touch-icon, og-image, Chrome Web Store tiles) from one source SVG via [`@resvg/resvg-js`](https://github.com/yisibl/resvg-js).
- `brandkit gen` ‚Äî full pipeline: provider call (fal.ai Ideogram v3) ‚Üí vectorize (Recraft) ‚Üí recolor ‚Üí verify ‚Üí gallery for human selection.
- MCP server (`mcp/`) exposing `brandkit_recolor`, `brandkit_verify`, `brandkit_gen` over stdio JSON-RPC.
- Pluggable provider interface: `fal` and `replicate` (stub) for image generation; `recraft` and `vectorizer` (stub) for vectorization.

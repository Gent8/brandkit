# Project status

**brandkit is solo-maintained, best-effort, no SLA.**

What that means in practice:

- **Issues** — read on a weekly cadence. Reproducible bugs get acknowledged within a week. Feature requests may sit indefinitely; consider opening a PR instead.
- **PRs** — small, focused PRs that include a test get reviewed first. Sweeping refactors will be closed unless discussed in an issue first.
- **Breaking changes upstream** (Ideogram / Recraft / fal.ai API changes) — fixed when noticed, not on a schedule. PR welcome if you hit one before I do.
- **Roadmap** — there isn't one. brandkit is a tool I built to scratch my own itch and chose to share. New features land when I want them.
- **Security** — see [SECURITY.md](SECURITY.md) if added; otherwise email the maintainer via GitHub profile.

If your team needs a guaranteed-maintained version, fork it. The code is intentionally small (~700 lines of JS).
